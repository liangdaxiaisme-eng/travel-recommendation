# 飞猪旅行推荐系统 - NeuMF版本

## 📋 系统简介

这是一个基于 **NeuMF（神经协同过滤）深度学习模型**的旅游景点推荐系统。

### 技术栈

- **后端框架**：Flask
- **AI 模型**：PyTorch NeuMF（神经网络协同过滤）
- **模型权重**：真实训练的权重文件（13MB）
- **数据来源**：真实用户行为数据

### 系统特点

✅ **真实 PyTorch 模型** - 使用训练好的 NeuMF 权重
✅ **深度学习推荐** - 基于 NeuMF 神经网络模型
✅ **个性化推荐** - 根据用户历史行为生成推荐
✅ **实时计算展示** - 右侧终端显示 NeuMF 计算过程
✅ **城市名称显示** - 自动识别景点所在城市
✅ **访问次数统计** - 显示每个景点的历史访问次数

## 🧠 NeuMF 模型说明

### 模型架构

NeuMF（Neural Collaborative Filtering）是一种深度学习推荐模型，结合了：

1. **GMF 分支**（矩阵分解）
   - 用户嵌入：8维
   - 物品嵌入：8维
   - 乘法融合

2. **MLP 分支**（多层感知机）
   - 用户嵌入：64维
   - 物品嵌入：64维
   - 知识图谱嵌入：32维
   - 层结构：64 → 32 → 16

3. **融合输出**
   - GMF + MLP 特征拼接
   - 输出层：1维（预测分数）

### 模型参数

- **用户数量**：63,161
- **物品数量**：69,525
- **知识图谱节点**：65
- **模型收敛 Loss**：0.4766

### 权重文件

**文件**：`neumf_weights_real.pt`（13MB）

这个权重文件是使用真实数据训练得到的，包含：
- 用户嵌入矩阵
- 物品嵌入矩阵
- 知识图谱嵌入矩阵
- MLP层权重

## 🐛 常见问题

### Q1: 系统启动慢怎么办？

**原因**：加载10万条数据需要时间（约3-5分钟）

**解决方案**：
1. 减少数据加载量（修改 `nrows` 参数）
2. 使用 SSD 存储
3. 增加服务器内存

### Q2: 权重文件找不到？

**错误信息**：`FileNotFoundError: neumf_weights_real.pt`

**解决方案**：
1. 确保权重文件在同一目录下
2. 检查文件名是否正确
3. 检查文件权限

### Q3: 推荐结果不准确？

**可能原因**：
1. 用户ID不存在
2. 用户没有历史记录
3. 数据量太少

**解决方案**：
1. 使用测试用户ID（1000560、1000620）
2. 确保用户有历史记录
3. 增加数据加载量

### Q4: 端口被占用？

**错误信息**：`Address already in use`

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i:4009

# 杀掉进程
kill -9 <PID>

# 或者使用其他端口
python3 recommendation_neumf_fixed.py
```

### Q5: 如何停止服务？

**方法1**：Ctrl + C（在终端中）

**方法2**：查找进程并杀掉

```bash
ps aux | grep recommendation_neumf_fixed.py
kill -9 <PID>
```
