## 🎯 需要的数据类型

### 1. 景点数据（使用八爪鱼采集器采集）

**核心字段：**
- 景点 ID（唯一标识）
- 景点名称
- 景点描述（文本，用于 NLP 处理）
- 地理位置（经纬度，用于地理推荐）
- 城市/地区
- 评分（1-5 分）
- 评论数
- 景点类型（自然风光/人文历史/主题公园/其他）
- 票价
- 开放时间
- 图片 URL（多模态数据）
- 标签/关键词

**数据来源推荐：**
1. **高德地图 API** - POI 数据、地理位置
2. **百度地图 API** - 景点信息、评论
3. **携程旅行 API** - 景点详情、用户评价
4. **马蜂窝旅游网** - 游记、攻略、图片
5. **美团旅游** - 门票、评价数据

### 2. 用户数据

**核心字段：**
- 用户 ID（匿名化）
- 人口统计信息（年龄、性别、城市）
- 用户偏好（喜欢的景点类型）
- 历史行为（浏览、收藏、评分、评论）
- 地理位置信息

### 3. 交互数据

**核心字段：**
- 用户 - 景点评分矩阵
- 用户访问记录（时间戳、停留时长）
- 用户评论（文本、评分、时间）
- 用户行为（点击、收藏、分享）

## 🔧 数据采集方法

### 方法 1：使用八爪鱼采集器（推荐）

**步骤：**
1. 下载并安装八爪鱼采集器：https://www.octoparse.com/
2. 打开八爪鱼，选择"自定义采集"
3. 输入目标网站 URL（如马蜂窝、携程）
4. 使用智能识别功能自动识别景点列表
5. 配置采集字段（参考 `数据采集模板.json`）
6. 设置爬取间隔（3-5 秒/次，避免被封）
7. 运行采集并保存为 JSON/CSV 格式
8. 将数据保存到 `/原始数据/` 目录

**注意事项：**
- ⚠️ 遵守网站 robots.txt 协议
- ⚠️ 控制爬取频率
- ⚠️ 不采集个人隐私数据
- ⚠️ 仅用于学术研究

## 🧹 数据预处理流程

### 1. 数据清洗
- 去除重复数据
- 处理缺失值（删除或填充）
- 统一数据格式（日期、价格等）
- 去除特殊字符和无关内容

### 2. 特征工程
- **地理位置编码：** 经纬度转换为网格 ID 或使用地理哈希
- **文本向量化：** 
  - TF-IDF
  - Word2Vec
  - BERT 嵌入
- **用户画像构建：** 基于历史行为生成用户特征向量
- **多模态特征提取：**
  - 图像：使用 CNN（ResNet、VGG）提取视觉特征
  - 文本：使用 Transformer 提取语义特征

### 3. 数据划分
- 训练集：70%
- 验证集：15%
- 测试集：15%

### 4. 数据标准化
- 评分归一化到 [0, 1]
- 价格归一化
- 文本长度截断/填充

## 🤖 模型架构（根据开题报告）

### 核心模型：多模态数据融合模型

**架构设计：**
```
输入层：
├── 用户特征（人口统计、历史行为）
├── 景点特征（文本描述、图片、位置、类型）
└── 上下文特征（时间、天气、季节）

特征融合层：
├── 特征拼接/相加
└── 模态注意力机制（动态加权各模态）

深度学习层：
├── 多层感知机 (MLP)
├── 注意力网络
└── 图神经网络 (GNN) - 处理用户 - 景点关系

输出层：
└── 推荐评分预测 / 排序分数
```

### 解决的核心问题

1. **数据稀疏性：**
   - 引入多源辅助数据（用户侧 + 景点侧）
   - 使用知识图谱增强特征
   - 迁移学习从其他领域迁移知识

2. **冷启动问题：**
   - 基于内容的推荐（利用景点描述、图片）
   - 迁移学习（从其他城市/场景迁移）
   - 探索 - 利用策略（Epsilon-Greedy）

3. **推荐多样性：**
   - 多目标优化（精度 + 多样性）
   - 用户画像分群（差异化推荐）
   - 强化学习动态调整策略

## 📈 评估指标

- **准确率：** Precision@K
- **召回率：** Recall@K
- **F1 分数：** F1@K
- **RMSE：** 均方根误差（评分预测）
- **NDCG：** 归一化折损累计增益（排序质量）
- **多样性：** 推荐列表的多样性得分
